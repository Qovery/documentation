## Overview

View and stream logs from any service â€” application, container, database, or job â€” running on Qovery.

## Commands

### Stream Logs

Stream real-time logs:

```bash
qovery log
```

Stream logs for a specific service (regardless of its type):

```bash
qovery log --service "httpbin"
```

Follow logs with live updates:

```bash
qovery log --service "httpbin" --follow
```

Filter logs by keyword:

```bash
qovery log --service "httpbin" --filter "ERROR"
```

### Historical Logs

Get logs from last hour:

```bash
qovery log --service "httpbin" --since 1h
```

Get logs from specific time range:

```bash
qovery log \
  --service "httpbin" \
  --from "2024-01-01T00:00:00Z" \
  --to "2024-01-01T23:59:59Z"
```

Get last N lines:

```bash
qovery log --service "httpbin" --tail 100
```

## Options

| Flag            | Description                                                              |
| --------------- | ------------------------------------------------------------------------ |
| `--service`     | Service name â€” works for any type (application, container, database, job) |
| `--application` | Application name (legacy, prefer `--service`)                            |
| `--container`   | Container name (legacy, prefer `--service`)                              |
| `--database`    | Database name (legacy, prefer `--service`)                               |
| `--job`         | Job name (legacy, prefer `--service`)                                    |
| `--follow`      | Follow log output                                                        |
| `--filter`      | Filter logs by keyword                                                   |
| `--since`       | Show logs since duration (e.g., 1h, 30m)                                |
| `--from`        | Show logs from timestamp (ISO 8601)                                      |
| `--to`          | Show logs until timestamp (ISO 8601)                                     |
| `--tail`        | Number of lines to show from end                                         |
| `--help`        | Show help                                                                |

## Examples

### Follow Service Logs

```bash
# Stream logs in real-time â€” works for any service type
qovery log --service "httpbin" --follow

# Filter for errors
qovery log --service "httpbin" --follow --filter "ERROR"
```

### Debug Production Issues

```bash
# Get last hour of logs
qovery log \
  --service "my-api" \
  --since 1h \
  --filter "exception"

# Get specific time range
qovery log \
  --service "my-api" \
  --from "2024-12-01T10:00:00Z" \
  --to "2024-12-01T11:00:00Z"
```

### View Database Logs

```bash
# Stream database logs â€” no need to specify the type
qovery log --service "postgres-main" --follow
```

### View Job Logs

```bash
# View cronjob logs
qovery log --service "backup-job" --tail 50
```

### Multiple Filters

```bash
# Combine flags for precise debugging
qovery log \
  --service "my-api" \
  --since 30m \
  --filter "ERROR" \
  --tail 100
```

## Tips

<Tip>
  Use `--service` with just the service name â€” Qovery automatically resolves its
  type (application, container, database, job). No need to know the type upfront.
</Tip>

<Tip>
  Use `--follow` to monitor logs in real-time during deployments or debugging.
</Tip>

<Tip>
  Combine `--filter` with error levels like "ERROR", "WARN", "FATAL" to quickly
  find issues.
</Tip>

<Tip>
  The `--since` flag accepts: `s` (seconds), `m` (minutes), `h` (hours), `d`
  (days).
</Tip>
