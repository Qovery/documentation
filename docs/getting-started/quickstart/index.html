Quick Start Guide

To quickly learn how to use Qovery, we recommend you to follow our Getting Started Guide, which covers all the most important concepts you may need while developing your applications with Qovery.

Getting Started Guide

If you don't want to learn more now, you can simply deploy your first application following just three steps from below and get back to the Getting Started Guide later on.

#Sign Up

#Install Qovery CLI

  • Linux
  • MacOS
  • Windows
# Download and install Qovery CLI on every Linux distribution
$ curl -s https://get.qovery.com | sudo bash

#Sign up

# Sign up and sign in command
$ qovery auth

Your browser window with sign-in options will open.

Note: Qovery needs access to your account to be able to clone your repository for future application builds.

Click here to authorize Qovery to clone and build your applications.

Congratulations, you are logged-in.

#Create an Application

To show you how it's simple to build and deploy your applications on Qovery, we have prepared a sample project.

#Fork a sample application

Fork a sample application by clicking on the Fork button:

After it's done, clone your forked repository:

git clone git@github.com:<your-github-nickname>/simple-example.git
cd simple-example

In this repository, you'll find an already existing Dockerfile able to build and run the application.

#Initialize Qovery

In order to be able to deploy this application using Qovery, you have to bootstrap the configuration file:

$ qovery init

We need to get the access to your Git repository to be able to build a Docker image container and access your .qovery.yml configuration file.

You'll have to grant Qovery (only the first time) access to your repository:

Then, you'll see Qovery access permissions:

You can grant Qovery access to all repositories or just those you use with Qovery.

#Deploy!

After you have initialized Qovery configuration and gave the permissions to access your repository, all you have to do to deploy your application is to commit and push your changes:

git add .qovery.yml
git commit -m "Adding first Qovery configuration" .qovery.yml
git push

That's it! We'll detect changes in your repository and build and deploy your application.

#Check the deployment status

To check the status of your application, run qovery status command:

qovery status

#Push new changes

From now on, any changes in your remote Git repository will trigger a CI/CD pipeline that will build and deploy your application. Enjoy!
